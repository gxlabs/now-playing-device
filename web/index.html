Now Playing Display

A macOS menu bar app that streams what you're playing — album art, track info, progress — to a USB-connected ESP32-C6 round display.

macOS 13+ Apple Silicon & Intel USB-C

Download

Download NowPlayingDisplay.zip

How it works

┌───────────┐ USB serial ┌────────────────┐ /usr/bin/python3 ┌──────────────────┐ │ ESP32-C6 │ ◄──────────── │ Menu bar app │ ◄──────────────── │ Apple Music │ │ + display │ ─────────────► │ (Mac side) │ ─────────────────► │ / Spotify, etc │ └───────────┘ touch cmds └────────────────┘ media controls └──────────────────┘

The Mac reads now-playing metadata via the bundled MediaRemoteAdapter.framework, loaded in-process by /usr/bin/python3 (a com.apple.*-signed binary — macOS 15.4+ only authorizes the private MediaRemote framework for callers with an Apple bundle id, so the bundled py2app Python can't reach it directly). Artwork is converted to RGB565 and pushed to the ESP32 over USB; touches on the round display send prev/play-pause/next back to the Mac.

Hardware

  • Seeed XIAO ESP32-C6
  • Seeed Round Display for XIAO — 240×240 GC9A01A + CHSC6X touch
  • USB-C cable

Flash the firmware

Pre-built firmware is available as a single merged image: firmware.bin (flash to offset 0x0).

Install esptool if you don't have it:

pip install esptool   # or: brew install esptool

Plug in the XIAO ESP32-C6 over USB-C, then flash:

esptool.py --chip esp32c6 --port /dev/cu.usbmodem* write_flash 0x0 firmware.bin
Note — on Linux/Windows the port path is different (/dev/ttyUSB0, COM3, etc). If esptool can't enter download mode automatically, hold the BOOT button on the XIAO while plugging in (or pressing RESET).

Or build from source via ESP-IDF (idf.py build flash) using the firmware/ tree in the repo.

Set up the Mac app

Drag NowPlayingDisplay.app to /Applications and launch it. The app auto-detects the ESP32 over USB on every tick — plug the device in any time and it'll just connect.

Menu bar

  • Circle with a checkmark when connected, cross when not
  • Dropdown shows the USB port path and the current track
  • "Hide" removes the menu bar slot — the bridge keeps running in the background; relaunch the app to bring the icon back
